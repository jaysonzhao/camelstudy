Additional Enrichment and Aggregation on Red Hat Fuse Online Lab

In this lab, you extend the integration created in the previous lab in Red Hat® Fuse Online to expose the integration as a REST API. You develop the solution using the console in Red Hat Fuse Online.

Expect your instructor to provide you with a Red Hat Fuse Online console URL with this format: https://user$seq-fuse-online.$OCP_DOMAIN.

Goals

  • Design a Red Hat Fuse Online integration that consumes JSON records from a REST API

  • Enrich accounts using a back-end REST service

  • Further enrich accounts using a back-end SOAP service with a REST wrapper

  • Map the responses from the back-end services to the data fields

  • Save the aggregated message in an SQL database

  • Send a response to the API consumer

    Red Hat Fuse Online REST Enrich Integration

1. Edit Integration

In this section, you create a new integration as shown in the integration flow diagram in the introduction to this lab.

  1. Using the URL provided by your instructor and specifying your Red Hat OpenShift® Container Platform user ID and password, log in to the console in Red Hat Fuse Online.

  2. Click Create a New Integration.

  3. Choose API Provider as the start connection.

  4. Choose to create from scratch.

    • Expect to be presented with an Apicurito design page.

  5. On the Apicurito design page, enter the following details:

    • Name: AccountUploadAPI

    • Path: /rest/account

    • Operation: PUT

    • Data Type: Account

    • Data Type Example: 

      {"company":{"name":"Rotobots","geo":"NA","active":true},"contact":{"firstName":"Bill","lastName":"Smith","streetAddr":"100 N Park Ave.","city":"Phoenix","state":"AZ","zip":"85017","phone":"602-555-1100"}}

    • PUT Summary: Create Account Enrich and Upload record

    • Consumes (inputs): application/json

    • Produces (outputs): application/json

    • Request Body Type: Account

    • Response: 200 OK

    • Response Type: String

    • Response Example: 

      {"result": "Account created successfully."}

2. Publish and Test API

  1. Save and publish the integration.

    • Expect the URL to the API to be available from the integration page.

  2. Access the REST API specification by opening the <i-apitosqlenrich route>/openapi.json URL.

  3. Send a curl request to the API: 

    curl -k <i-apitosqlenrich route>/rest/account -X PUT  -d '{"company":{"name":"Rotobots","geo":"NA","active":true},"contact":{"firstName":"Bill","lastName":"Smith","streetAddr":"100 N Park Ave.","city":"Phoenix","state":"AZ","zip":"85017","phone":"602-555-1100"}}' -H 'content-type: application/json'

  4. Verify that you see a response similar to this: 

    {"result": "Account created successfully."}

  5. Examine the integration activity log for errors and the T_ACCOUNT SQL table to make sure the request processed successfully.
