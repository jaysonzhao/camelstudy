Integration with Red Hat 3scale API Management Lab

In this lab, you change the integration API for Red Hat® Fuse Online to allow service discovery by 3scale API Management. You then manage the API in 3scale API Management and make sure that the requests are routed to the integration through the 3scale API Management gateway.

Goals

  • Change the integration service to make it discoverable

  • Test discovery of the integration service in 3scale API Management

  • Create an application plan and application in 3scale API Management

  • Test the API request via 3scale API Management

1. Edit Service

In this section, you edit the i-apitosqlenrich service created earlier in this course to make the service discoverable by 3scale API Management. This ensures that 3scale API Management can automatically scan the services in the Red Hat OpenShift® Container Platform cluster and that the service is imported correctly.

  1. Edit the i-apitosqlenrich service in the OpenShift user$seq-fuse project: 

    oc edit svc i-apitosqlenrich

  2. Add the following line under labels: 

        discovery.3scale.net: "true"

  3. Add the following lines under annotations: 

        discovery.3scale.net/description-path: /openapi.json
    discovery.3scale.net/path:
    discovery.3scale.net/port: "8080"
    discovery.3scale.net/scheme: http

  4. Save the changes.

2. Use 3scale API Management

In this section, you complete the remainder of the configuration in the 3scale API Management Platform administration console.

2.1. Log in to Administration Console

  1. Log in to your 3scale API Management Platform administration console with the user ID and password details provided by the instructor.

2.2. Test Discovery

  1. On the home page, click NEW API:

  2. Click Import from OpenShift and select your Red Hat Fuse Online namespace and corresponding i-apitosqlenrich service:

  3. Click Create Service.

    • Within a few minutes, expect the service to be imported and appear in the list of integrations.

2.3. Integrate API

  1. Next to the API service name, click INTEGRATE THIS API:

  2. Provide the Staging Public Base URL and Production Public Base URL corresponding to the OpenShift route to your apicast-staging and apicast-production applications.

  3. Add a new mapping rule using these values:

    • Verb: PUT

    • Pattern: /

    • +: 1

    • Metric or Method (Define): hits

  4. Keep the default values unchanged, then click Update & test in Staging Environment.

  5. Click Create Application Plan and enter the following values:

    • Name: basic

    • System name: i-apitosqlenrich/basic

  6. Click Create Application Plan to create the plan.

  7. Click Publish to publish the application plan.

2.4. Create Application

  1. Click Audience.

  2. Under Apps for the Developer account, click 1:

  3. Click Create Application:

  4. On the Create Application page, select the following values:

    • Application Plan: basic (same application plan as you created in the integration)

    • Name: DeveloperAccountPlan

    • Description: Developer Account Plan

  5. Click Create Application.

  6. Make note of your API Credentials user key, as you need this to make a request to the API on 3scale API Management:

3. Test Staging Route

  1. Make a request to the 3scale API Management staging route, replacing <user$seq staging route> with the appropriate staging public base URL to your service and <user$seq application key> with the key from the previous step: 

    curl -k <user$seq staging route>/rest/account?user_key=<user$seq application key> -X PUT  -d '{"company":{"name":"Rotobots","geo":"NA","active":true},"contact":{"firstName":"Bill","lastName":"Smith","streetAddr":"100 N Park Ave.","city":"Phoenix","state":"AZ","zip":"85017","phone":"602-555-1100"}}' -H 'content-type: application/json'

  2. Expect to receive the following response if the request is successful:

    Sample Output
    {"result": "Account created successfully."}

  3. Examine the 3scale API Management analytics to make sure that the request is recorded.

  4. Examine the integration activity log for Red Hat Fuse Online to monitor the request.

4. Complete Bonus Exercise

  1. Promote the service to production in 3scale API Management.

  2. Verify that the API request to production works successfully.
