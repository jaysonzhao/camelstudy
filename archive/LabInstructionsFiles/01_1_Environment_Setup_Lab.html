Environment Setup Lab

In this lab, you set up the environment for the labs in this course.

Goals

  • Set up the developer tools on your workstation

  • Authenticate the OpenShift client with the Red Hat® OpenShift® Container Platform master API

  • Install and use the OpenShift command line interface (CLI)

Prerequisites

  • An open broadband Internet connection

  • A terminal client, such as PuTTY for Microsoft Windows

  • A current browser, such as Google Chrome or Mozilla Firefox

  • Your OPENTLC credentials

  • Sufficient disk space to install Red Hat Developer Studio version 12.9.0, Red Hat Fuse, Red Hat AMQ, and other programming tools (Maven, OpenShift CLI, Java™)

    Important
    		 Make sure that the Internet connection is free of corporate proxies or other network rules that prevent access to remote servers on the Internet.

1. Install Required Software

In this lab, you create Red Hat Fuse on OpenShift applications, using Red Hat Developer Studio and OpenShift CLI tools on the workstation, and deploy them to an OpenShift project.

  1. Install the software listed here on your local machine:

    • Java SE^ (version 1.8)

    • Apache Maven^ (version 3.3.9+)

    • Git^ (latest version)

    • OpenShift CLI client^ (version 3.11+)

    • (Optional) SoapUI^ (latest version)

To complete the labs in the course, you must have Red Hat Developer Studio installed in your local development environment. You use Red Hat Developer Studio to design Apache Camel routes.

  3. Proceed to log in.

2.2. Review Red Hat Developer Studio Integration Stack Plug-ins

Red Hat Developer Studio includes a variety of plug-ins for Eclipse. You use the following Red Hat Developer Studio plug-ins to complete the labs in Red Hat OPEN middleware courses:

  • Integration Stack: The Integration Stack suite of plug-ins is particularly important when using Red Hat Fuse and Red Hat AMQ. The Integration Stack suite is now included with Red Hat Developer Studio. No additional installation is required

  • EGit: Red Hat Developer Studio includes the Eclipse EGit plug-in, which provides Git project support. No additional installation is required. Git is an open source version control system, providing developers with fast, versatile access to their application code’s entire revision history.

  • M2E: Red Hat Developer Studio includes the Eclipse M2E plug-in, which provides support for Apache Maven projects. No additional installation is required. The M2E plug-in enables you to edit a Maven projects pom.xml and run a Maven build from the IDE.

6. Prepare OpenShift Project

A workshop OpenShift cluster is provisioned for use during the class. The GUID and login information is provided by your instructor.

6.1. Create OpenShift Project

Now that you have provisioned the lab environment, you can log in to the OpenShift Container Platform web console and create a project.

  1. In a browser, navigate to the URL of the OpenShift shared cluster environment provided to you—for example, https://master.na311.openshift.opentlc.com.

    Important

    Pay close attention to the region code. In this example, the region code is na311, but your region may be different.

    Use Google Chrome, Mozilla Firefox, or Microsoft Internet Explorer to access the OpenShift Container Platform web console. The Safari browser does not display the page correctly.

  2. Provide your username and password to log in to the OpenShift Container Platform web console.

  3. Create a new project for deploying your Red Hat Fuse on OpenShift applications, entering the following in the New Project dialog:

    • Name: $OCP_PROJECT_PREFIX-fuse

    • Display Name: Fuse Experienced Project

      Note
      		 $OCP_PROJECT_PREFIX is a unique prefix for your project. You can set it to your initials or the user ID. Because this prefix is used for all of the projects you create, be sure to make a note of the prefix you use.

  4. Click Create.

6.2. Authenticate OpenShift Client

The OpenShift Container Platform environment is ready for you to install OpenShift applications. You interact with this environment via the OpenShift CLI or the OpenShift Container Platform web console. Before you can use the OpenShift client, you must authenticate with the OpenShift master API.

  1. On your workstation, open a terminal shell using—for example, PuTTY for Windows.

  2. Authenticate with the OpenShift master API, replacing the value after the -u parameter with your Red Hat OPENTLC username, and the value after //master. with the region specified in your provisioning email: 

    $ oc login https://master.$REGION.openshift.opentlc.com:443 -u $USERID
    Sample Output
    Authentication required for https://master.na311.openshift.opentlc.com:443 (openshift)

Login successful.

6.3. Deploy AMQ Broker on OpenShift Container Platform

In this section, you deploy AMQ Broker 7.2 on OpenShift Container Platform. The general installation steps are included in the Deploying AMQ Broker on OpenShift Container Platform^ documentation.

  1. In a terminal shell, log in to the OpenShift Container Platform cluster.

  2. Change to the $OCP_PROJECT_PREFIX-fuse OpenShift project: 

    $ oc project $OCP_PROJECT_PREFIX-fuse

  3. Add the view role to the service account: 

    $ echo '{"kind": "ServiceAccount", "apiVersion": "v1", "metadata": {"name": "amq-service-account"}}' | oc create -f -
$ oc policy add-role-to-user view system:serviceaccount:$OCP_PROJECT_PREFIX-fuse:amq-service-account

  4. Deploy the broker: 

    $ oc new-app --namespace $OCP_PROJECT_PREFIX-fuse \
   --template=amq-broker-72-basic \
   -e AMQ_PROTOCOL=openwire,amqp,stomp,mqtt,hornetq \
   -e AMQ_USER=admin \
   -e AMQ_PASSWORD=password \
   -e AMQ_ROLE=admin

  5. Verify that the broker pod is running.

  6. Make a note of the console URL.

  7. Navigate to the console URL from a web browser and make sure that the broker is started correctly.

  8. Make a note of the broker service URL for AMQP, broker-amq-amqp:5672.

    • This URL is required for configuring the AMQ connection later in the labs.

6.4. Deploy PostgreSQL on OpenShift Container Platform

In this section, you deploy the PostgreSQL database on OpenShift Container Platform.

You can find more detailed installation instructions in the PostgreSQL on OpenShift^ guide.

  1. In a terminal shell, log in to the OpenShift Container Platform cluster.

  2. Change to the $OCP_PROJECT_PREFIX-fuse OpenShift project: 

    $ oc project $OCP_PROJECT_PREFIX-fuse

  3. Deploy the database: 

    $ oc new-app --namespace $OCP_PROJECT_PREFIX-fuse \
    -e POSTGRESQL_USER=postgres \
    -e POSTGRESQL_PASSWORD=postgres \
    -e POSTGRESQL_DATABASE=sampledb \
    postgresql-persistent

  4. Verify that the database pod is running.

  5. Identify the name of the pod running PostgreSQL and open a remote shell to the pod: 

    $  oc get pods | grep postgresql
$  oc rsh <pod>

  6. Create the sampledb database, logging in as the postgres user: 

    sh-4.2$ createdb -h localhost -p 5432 -U postgres sampledb

  7. Start the psql shell: 

    sh-4.2$ PGPASSWORD=$POSTGRESQL_PASSWORD psql -h postgresql $POSTGRESQL_DATABASE $POSTGRESQL_USER
psql (9.6.10)
Type "help" for help.

sampledb=#

  8. Create the tables from within the psql shell: 

    CREATE SCHEMA USECASE;
CREATE TABLE USECASE.T_ACCOUNT (
    id  SERIAL PRIMARY KEY,
    CLIENT_ID integer,
    SALES_CONTACT VARCHAR(30),
    COMPANY_NAME VARCHAR(50),
    COMPANY_GEO CHAR(20) ,
    COMPANY_ACTIVE BOOLEAN,
    CONTACT_FIRST_NAME VARCHAR(35),
    CONTACT_LAST_NAME VARCHAR(35),
    CONTACT_ADDRESS VARCHAR(255),
    CONTACT_CITY VARCHAR(40),
    CONTACT_STATE VARCHAR(40),
    CONTACT_ZIP VARCHAR(10),
    CONTACT_EMAIL VARCHAR(60),
    CONTACT_PHONE VARCHAR(35),
    CREATION_DATE TIMESTAMP,
    CREATION_USER VARCHAR(255)
);
CREATE TABLE USECASE.T_ERROR (
    ID SERIAL PRIMARY KEY,
    ERROR_CODE VARCHAR(4) NOT NULL,
    ERROR_MESSAGE VARCHAR(255),
    MESSAGE VARCHAR(512),
    STATUS CHAR(6)
);
    Tip
    		 You can use \q to exit the psql shell.

Your environment is now set up correctly and you can proceed with the rest of the labs.
