Red Hat AMQ 7

What Is New

  • Consolidation of Red Hat messaging products to create new best-of-breed offering

    • Red Hat Enterprise MRG Messaging (based on Apache Qpid C++ broker)

    • Red Hat A-MQ 6.x (based on ActiveMQ Java™ broker)

    • Red Hat JBoss® Enterprise Application Platform 6.x messaging (based on HornetQ)

  • Focus on international standard protocols, AMQP and MQTT

  • Multiple AMQP-based client libraries including Ruby, JMS 2.0, C++, .NET, Python and JavaScript (with Node.js)

  • New router component provides global routing with direct message delivery

  • Red Hat AMQ Streams will provide a supported offering for running Apache Kafka on
    Red Hat Enterprise Linux    ® and on Red Hat OpenShift® Container Platform

  • Red Hat Managed, self-service messaging in the cloud through Red Hat AMQ Online

Red Hat AMQ 7

Deployment Overview

Red Hat AMQ 7

Broker

  • Core messaging provided by Artemis broker in the following styles

    • Publish-subscribe

    • Point-to-point

    • Store-and-forward

  • Multiple protocols and client languages supported

    • Java JMS

    • .NET

    • Node.js

    • Python

    • C++

  • Fully asynchronous (non-blocking) architecture

  • Thread pooling

  • High-performance journal

Red Hat AMQ 7

Interconnect Router

  • AMQP-native message router

  • Influenced by Red Hat MRG Messaging use cases

  • Supports high performance direct messaging

  • 1-to-1 or 1-to-many

  • Network offers shortest-path routing with redundancy

  • Can be used standalone or in conjunction with broker

  • Only available on Red Hat Enterprise Linux

Red Hat AMQ 7

Router Implementation

  • A network service (a messaging intermediary)

  • Implemented in C and embedded Python

  • Asynchronous/event-driven architecture

  • Designed for high throughput and low latency

  • Intended to be deployed in networks of arbitrary topology

  • High availability through redundant paths, not state replication

  • Convergence of networking and messaging

     

Red Hat AMQ Online

  • Open source messaging platform on OpenShift Container Platform and Kubernetes

  • Messaging built around address spaces and addresses

  • Project goals

    • Scaling and performance

    • Simple to deploy and manage

    • Multitenancy using multiple address spaces

    • Deploy on-premises or in the cloud

    • Open standard support, such as AMQP and MQTT

Red Hat AMQ Online

Address Model

Red Hat AMQ Online

Address Space and Addresses

  • Address Space

    • Group of addresses accessible through single connection

    • Type: Defines the infrastructure to be deployed

  • Address

    • Type: Defines semantics of address

    • Plan: Affects what infrastructure is used or deployed for this address

    • Has set of properties

    • Available address types determined by the address space type

    • Available plans determined by the address type

    • Plan: May define quota restrictions (for example, maximum number of addresses)

    • Available plans determined by the type of address space

Red Hat AMQ Online

Address Types

  • Focused on scaling number of connections and throughput

  • Supports AMQP and MQTT protocols

  • Based on Apache Artemis and Apache Qpid Dispatch router projects

  • Supports four address types:

    • Queue: Store-and-forward semantics

    • Topic: n:m publish and subscribe

    • Anycast: Direct address for sending to one consumer

    • Multicast: Direct address for sending to multiple consumers

Red Hat AMQ Streams

  • Publish-subscribe messaging system

  • Streaming data platform

  • Distributed, horizontally-scalable, fault tolerant commit log

  • Capable of message replay

  • Architecture enables very high messaging rates

  • Use cases:

    • Web site activity tracking

    • Operational monitoring of data, aggregation of statistics

    • Event sourcing

    • Stream processing

    • Log aggregation

Red Hat AMQ Streams

  • Distributed backbone that allows microservices and other applications to share data with extremely high throughput and extremely low latency

  • Optimized for running Apache Kafka on Red Hat OpenShift Container Platform

  • Upstream project: Strimzi

  • Provides:

    • Container images for running Apache Kafka and Apache ZooKeeper

    • Operators for creating, managing, and configuring Kafka clusters, topics, and users

Red Hat AMQ Streams

  • High availability

  • Metrics

Red Hat AMQ Streams

Concepts

  • Messages are published to, and consumed from, topics

    • Topics are split into partitions

    • Subscribers consume from one or more topics

    • Message subscription at partition level

  • Each message written to a single partition only

    • Partitioning done on message key

    • Message order guaranteed within a partition

  • Message retention

    • ​Based on message size and message key

    • Ability to compact messages for a key

Kafka Streams

  • Stream processing framework

  • Streams are Kafka topics (as input and output)

  • Java library to include in your application

  • Creates a topology of processing nodes (filter, map, join) acting on a stream

    • Low level processor API

    • High level DSL

    • Using “internal” topics (when repartitioning is needed, or for “stateful” transformations)
