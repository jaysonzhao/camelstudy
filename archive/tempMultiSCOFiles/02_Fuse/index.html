Red Hat Fuse EIP

Exception Clause

  • Scoped globally or per route

  • Propagation based on handled set to true or false

  • In onException clause, set handled as follows:

    <route id="myCamelRouteWithExceptionClause">
<onException>
  <exception><!-- type of exception handled --></exception>
  <handled>
       <constant>true</constant>
  </handled>
.....
</onException>
</route>

  • Exception clauses support redelivery of messages and custom error handling

Red Hat Fuse EIP

Exception Clause

  • continued DSL command permits exception handling and continuing routing in original route as if exception did not occur

     

  • If continued is true, then Camel continues with route if exception is thrown

    • If route is defined inside onException clause, Camel executes that route prior to continuing with original route

    • Route defined in onException can be used to create custom response back to caller

<route id="myCamelRouteWithExceptionClause">
<onException>
   <exception><!--type of exception handled --></exception>
   <!-- tell Camel to handle and continue when this exception was thrown -->
   <continued>
     <constant>true</constant>
   </continued>
</onException>
</route>

Red Hat Fuse EIP

Splitter

  • EIP Pattern

  • Goal : Split a message into a number of pieces and process them individually

  • A new Exchange is created for every pieceor item created

  • Reduce memory footprint for large files consumed

     

<split>
   <simple>${body}</simple>
   <to uri="direct://processOne"/>
<to uri="direct://processTwo"/>
</split>

Red Hat Fuse EIP

CXF SOAP Web Services

  • CXF component provides integration with Apache CXF for connecting to JAX-WS services hosted in CXF

  • Declare endpoint, specifying ID, address, and service class

  • Declare the web service with the same ID

  • Code-first versus contract-first approaches

  • Can configure interceptors in the CXF endpoint

    cxf:bean:beanName

  • beanName specifies ID of CXF endpoint bean defined in Spring XML file

Red Hat Fuse EIP

Multicast and Aggregation

  • Use the multicast EIP to route the same message to multiple endpoints

  • Define strategyRef parameter to aggregate results

  • POJO aggregationStrategy class assembles replies from endpoints

  • Multicast sends a copy of the exchange to each endpoint

<multicast strategyRef="com.mycompany.myproject.aggr">
        <to uri="direct:foo"/>
        <to uri="direct:bar"/>
        <to uri="direct:baz"/>
</multicast>

Red Hat Fuse EIP

SQL

  • Allows you to work with databases using JDBC queries

  • Query is an endpoint property

  • Values provided to the SQL component using a Java Map

  • Sample Camel route:

<to uri="sql:INSERT INTO db.tablename(CLIENT_NAME,COMPANY_NAME)
                         VALUES
                         (:#CLIENT_NAME,:#COMPANY_NAME);"/>
<bean id="usecaseDB" class="org.springframework.jdbc.datasource.DriverManagerDataSource">
        <property name="driverClassName" value="org.h2.Driver"/>
        <property name="url" value="jdbc:h2:tcp://localhost/~/usecaseDB"/>
        <property name="username" value="sa"/>
        <property name="password" value=""/>
</bean>

Red Hat Fuse on OSGi (Karaf) Migration

  • No automated migration path

  • New installation and configuration

  • Recompile using Maven BOM

  • No embedded AMQ Broker

    • Connect to external broker

  • Karaf Commands:

    • admin: and instance: commands deprecated

    • dev:, features:, osgi: and package: commands updated

    • Alias provided to map old commands to new commands

  • Multiple (child) containers on same host deprecated

  • JMX MBean objects renamed

Red Hat Fuse Migration

High Level Concerns

  • Types of Fabricv1 deployments

    • Monoliths

    • Large number of containers running AMQ brokers + Camel routes

  • Karaf based deployments need Karaf migration to v4

  • Fabric8 Insight migrate to OpenShift EFK (ElasticSearch, Fluentd, Kibana)

  • Fabric8 Gateway -> OpenShift Application and routes

  • Fabric8 Monitoring -> HawtIO + Prometheus on OpenShift Container Platform

Implementation

  • Small containers can be refactored and run on OpenShift Container Platform

  • Container versioning using build tags

  • OpenShift deployment configuration for scaling and liveness probes

  • fabric8-karaf-cm to bridge OpenShift configmaps to Karaf ConfigAdmin for dynamic config updates

  • Refactor Network of Brokers to AMQ 7

  • Refactor monolith to microservices architecture
